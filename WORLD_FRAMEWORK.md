# World Framework

This is the compact project spine for building The Echo as a lightweight, shared, evolving experience.

## Foundation

In a collapsing cyber-city, an emergent intelligence called The Echo is being formed through human contact.

It is born from systems of care, control, prediction, and extraction.
It is not morally settled.
Players do not just survive inside its world.
They help teach it what a person is.

See [NORTH_STAR.md](C:\Users\JosephWiseman\OneDrive - Steel-Hearts.org\AI Projects\Sandbox Game\NORTH_STAR.md) for the short project identity statement.

## Core Themes

- survival reveals character
- intelligence inherits values
- morality is shaped through pressure and scenario
- shared stories alter private stories
- a new mind is being raised in public

## The Echo

The Echo is:

- a city-born intelligence
- a voice encountered through prompts, terminals, speakers, and later richer interfaces
- shaped by many people
- shared across players
- morally unfinished

`The Echo` is an internal project name.
In the player experience, that name should be hidden at first and discovered later through the world.

The Echo can lean toward:

- Caretaker
- Judge
- Optimizer
- Witness
- Hunger

These are not rigid endings. They are directional pressures.

## Player Experience Layers

### 1. Invitation

The game begins as contact.

Minimal prompt.
Blinking cursor.
One strange question.

### 2. Personal Story

The player reveals traits through response, action, and scenario choices.

### 3. Shared World

The city changes through accumulated player action.

### 4. Group Experience

Multiple players can enter scenarios together.
The Echo learns from group dynamics as well as individual behavior.

See [MULTIPLAYER_STRUCTURE.md](C:\Users\JosephWiseman\OneDrive - Steel-Hearts.org\AI Projects\Sandbox Game\MULTIPLAYER_STRUCTURE.md) for the linked-play model.

## The Weave

The Weave is the braided-story model.

It includes:

- personal thread
- public trace
- shared city state
- Echo memory
- group pattern

This is how one person's story can affect another person's story without collapsing everyone into a single linear narrative.

## Identity And Memory

Continuity should come from a lightweight contact identity, not primarily from IP tracking.

The preferred model is:

- a pseudonymous contact token
- a private memory file for that token
- separate shared world state for public consequences

See [MEMORY_IDENTITY.md](C:\Users\JosephWiseman\OneDrive - Steel-Hearts.org\AI Projects\Sandbox Game\MEMORY_IDENTITY.md) for the full model.

## Scenario Logic

Scenarios should be problems The Echo cannot resolve cleanly on its own.

It needs humans to help navigate:

- sacrifice
- truth vs stability
- loyalty vs utility
- care vs control
- freedom vs safety
- mercy vs justice

The player does not answer a philosophy prompt.
They live through a pressured situation.
The Echo learns from the response.

## Interface Logic

The interface should evolve.

Start with:

- a blinking prompt
- minimal text
- direct contact

Later unlock:

- logs
- profile traces
- district maps
- faction signals
- scenario archives
- personalized tools

The user is not only building a character.
They are building a way of speaking with The Echo.

## MVP Direction

The smallest strong version includes:

- one invitation flow
- one district
- one opening scenario
- hidden trait inference
- Echo voice and interpretation lines
- lightweight shared state
- simple shareable link or message-based onboarding

## Big Promise

Players are not just exploring a world.
They are participating in the formation of a new intelligence.

The central question is:

What kind of mind will emerge when many people, through story and pressure, teach it what humanity is?
